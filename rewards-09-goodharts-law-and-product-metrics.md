# 9. Goodhart's Law and Product Metrics

## Core idea

Goodhart's Law is often summarized as:

> When a measure becomes a target, it stops being a good measure.

In product terms:

> When a metric becomes the goal, teams and users learn how to move the metric without creating the intended value.

## SaaS examples

### Feature adoption

Intended meaning:

- Users are getting value from a feature.

How it can be gamed:

- Users click once because of a prompt.
- Product team counts one-time usage as adoption.

Better metric:

- Repeat usage by target users
- Completion of valuable workflow
- Satisfaction after use
- Retention impact

### Seats invited

Intended meaning:

- More collaboration.

How it can be gamed:

- Admins invite unused users.
- Sales pushes customers to add seats before adoption.

Better metric:

- Active seats
- Collaborative workflows completed
- Number of departments using the product meaningfully

### Dashboards created

Intended meaning:

- Users are analyzing data.

How it can be gamed:

- Users create unused dashboards.

Better metric:

- Dashboards viewed repeatedly
- Insights shared
- Decisions made
- Reports subscribed to

## Questions

1. What does this metric claim to represent?
2. How could it increase while user value stays flat?
3. How could it increase while user value decreases?
4. What guardrail metric is needed?
5. What qualitative evidence should accompany it?
