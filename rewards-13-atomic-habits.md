# 13. Resource Summary: Atomic Habits

## Core idea

*Atomic Habits* explains habit formation through:

- Cue
- Craving
- Response
- Reward

And four design rules:

- Make it obvious
- Make it attractive
- Make it easy
- Make it satisfying

## SaaS application

Many SaaS products overuse reminders and underuse environment design.

Instead of adding more notifications, ask:

- Is the action obvious?
- Is it attractive?
- Is it easy?
- Is it satisfying?

## SaaS example: finance automation

Goal: users upload receipts weekly.

Bad design:

- Send repeated reminders.

Better design:

- Auto-detect receipts from inbox.
- Show missing items clearly.
- Make upload one-click.
- Give immediate confirmation.
- Show month-end close readiness improving.

## Questions

1. What cue should start the behavior?
2. Why would the user want to do it?
3. How can we reduce effort?
4. What immediate satisfaction follows?
5. What identity does the behavior reinforce?
