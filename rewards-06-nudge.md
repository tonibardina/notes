# 6. Resource Summary: Nudge

## Core idea

*Nudge* is about choice architecture: changing the environment so people are more likely to make better decisions while preserving choice.

In SaaS, many problems do not need rewards. They need better defaults, reduced friction, clearer feedback, or better timing.

## SaaS example: onboarding

A company wants more users to finish onboarding and considers giving credits.

A nudge-first approach might:

- Pre-fill known company data
- Recommend the most relevant template
- Connect to common integrations automatically
- Save progress
- Show why each step matters
- Delay advanced configuration until after first value

## SaaS example: admin settings

A security SaaS wants admins to enable two-factor authentication.

Reward approach:

- Give a badge for enabling 2FA.

Nudge approach:

- Make 2FA the recommended default.
- Show risk reduction.
- Offer a guided setup.
- Allow exceptions with explanation.
- Remind admins before risky events.

Combined approach:

- Show improved security maturity score after enabling 2FA.

## Questions

1. Are users unmotivated, or is the flow too hard?
2. What default would help most users?
3. What friction is accidental?
4. What friction is protective?
5. Could a nudge solve this better than a reward?

## Activity

Choose one low-conversion SaaS action:

- Connect integration
- Invite teammate
- Complete setup
- Create first report
- Configure automation
- Enable security setting
- Publish template

Design:

1. A reward-only intervention
2. A nudge-only intervention
3. A combined intervention

Then compare likely quality of behavior.
